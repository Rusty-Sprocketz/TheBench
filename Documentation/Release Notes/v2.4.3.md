# v2.4.3 — Fix "Unexpected token" JSON parse crash in pipeline-generated apps

## What Changed
- Added rule #16 to the Builder agent prompt requiring proper fetch error handling in generated apps
- Generated app.js must now check `response.ok` before calling `response.json()`, and handle non-JSON error responses (HTML 404/500 pages) gracefully with a user-friendly error message
- Prevents the "Unexpected token 'T', 'The page c...' is not valid JSON" crash

## Root Cause
Generated apps called `response.json()` directly after `fetch()` without checking `response.ok`. When the API endpoint returned an HTML error page (e.g. Vercel 404), the JSON parse failed with an unhandled exception.

## Files Modified
- `api/pipeline.js` (Builder prompt — new rule #16)
- `client/src/App.jsx` (version bump)
- `package.json` (version bump)
